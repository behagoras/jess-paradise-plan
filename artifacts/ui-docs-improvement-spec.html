Executive Summary

The highest-leverage move is to stop treating the wizard as the primary experience. The app's own docs argue for discovery-first: a live feed, vibe/budget/days filters, saved deals, alerts, and AI/editorial curation. The current UI still front-loads a long preference form before the user sees any trip.

UI Recommendations

Before: Form-First Funnel

Landing sells ease. Wizard immediately asks the user to review many chips and sliders. Summary repeats default state. Only then does discovery happen.

  • Keep: strong photography, clear tagline, provider-price caveat.
  • Change: make "Get started" open a short composer or live feed instead of full filter intake.
  • Risk: if all filters remain visible, the user may optimize the form rather than browse deals.

After: Discovery-First Funnel

First value is a trip card. The user can swipe/skip, favorite, save a filter, or open advanced filters only when needed.

  • Primary controls: budget, days, vibe, origin.
  • Secondary controls: package components, traveler type, intensity, weather, month.
  • Trust layer: "price seen X hours ago", "why this matched", and provider confirmation language.
