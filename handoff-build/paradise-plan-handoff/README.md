# Paradise Plan — Design & Experience Handoff Bundle

This bundle is everything a fresh Claude Code session needs to build the Paradise Plan
prototype: one interactive web app with **two independent verticals you can both see and switch**:

- **Design** (look & feel): 4 distinct visual identities, driven by editable design-token variables.
- **Experience** (flow paradigm): 4 distinct interaction flows over the same product.

Plus a live design-token panel (change the variables, change the design in real time).
Any Design can render any Experience: 16 coherent combinations. Ending is always the affiliate
model ("Reserve with [provider]"), no internal checkout. UI copy in English.

## How to use it

1. Open a new Claude Code session in an empty folder (or your repo).
2. **Copy the contents of `PROMPT.md` (full) or `PROMPT-SHORT.md` (brief) into the input box.**
3. Attach (or drop into the project folder) the files in this bundle so Claude Code can read them.
4. Let it read the inputs, propose its 4 designs and 4 experiences, and build the prototype.

## What's inside

- `PROMPT.md`         — full handoff prompt (copy into the box)
- `PROMPT-SHORT.md`   — short version of the same prompt
- `README.md`         — this file
- `flow/flujo-app-ui.md`        — THE app flow (primary input; Spanish, build in English)
- `flow/ParadisePlanMobile.pdf` — original mobile screens of the flow (the "flow PDF")
- `context/la-idea-de-negocio.md` — business idea / positioning / tone
- `context/prototipo-drift.md`    — prior design direction + constraints
- `context/drift-design-prompt.md`— prior design handoff prompt (craft reference)
- `context/Presentation.md`       — original problem, needs, personas
- `assets/screens/`               — original 2019 desktop UI mockups (reference only)
- `assets/Features-sitemap.jpg`   — feature map / sitemap
- `assets/Concept-map-journey.jpg`— 5-stage user journey

## Notes

- The reference docs are in Spanish; the prototype UI must be in **English**.
- The original screens are dated 2019 mockups: use them for **flow and content**, not the look.
- No em dashes in UI copy. No generic-AI-slop visuals.
